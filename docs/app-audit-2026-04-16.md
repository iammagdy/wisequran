# Wise Quran — Comprehensive App Audit & Remediation Plan
_Date: 16 April 2026_
_Scope: Performance, UI/UX, accessibility, offline/PWA, listening & recitation tools, hifz, prayer, Ramadan/Friday/azkar, stats, notifications, security, data integrity, code quality._

---

## 1. Executive summary

The app is a genuinely well-architected React 18 + Vite PWA. The bones are strong: routes are lazy-loaded, the audio player context is split three ways to avoid render storms, the virtualized Surah reader is in place, IndexedDB is versioned, the sync queue classifies retriable vs permanent errors, and PKCE auth + RLS are wired up. There is **no architectural rewrite required**.

What needs attention is a tier of **correctness, security, and reliability bugs** that quietly degrade the spiritual UX — wrong prayer times across DST, broken streaks at midnight/timezone changes, an admin route protected only by a hardcoded PIN, audio that may not resume from the lock screen, recitation matching that is unfair to Alef variants, and "backups" that don't include the offline content the user spent hours downloading.

Nothing here is on fire. Most items are 1–4 hour fixes. The plan at the end groups them into 4 phases so you can ship the high-impact changes first.

## 2. The short list — fix these first

| # | Area | Issue | Severity |
|---|---|---|---|
| 1 | Security | DevKit at `/devkit` is gated by a hardcoded PIN shipped in the JS bundle; once unlocked it can wipe all user data | **High** |
| 2 | Bookmarks | Anonymous local bookmarks are silently merged into a user account on sign-in — if two people share a device, data crosses accounts | **High** |
| 3 | Audio (iOS) | Lock-screen "Play" can fail to resume because the Media Session handler doesn't re-resolve the audio source if it was cleared during background suspension | **High** |
| 4 | Backups | Backup file does not include the actual offline Quran/audio/tafsir blobs — restoring on a new device looks "complete" but loses every download | **High** |
| 5 | Prayer times | Times for future dates are off by 1 hour around DST (timezone offset is read from "now" instead of the target date) | Medium |
| 6 | Adhan scheduling | Relies on `setInterval` + visibility events; backgrounded tabs (especially iOS) miss the actual prayer minute | Medium |
| 7 | Hifz streak | Streak calc uses millisecond floor — DST days break the streak | Medium |
| 8 | Recitation | Alef variants (أ إ آ) aren't normalized, so correct recitations are scored as wrong | Medium |
| 9 | Achievements | Reads `localStorage` directly inside a hook, so unlocks lag behind real progress until refresh | Medium |
| 10 | API caching | Quran/Tafsir API is `CacheFirst` for 30 days — text corrections won't reach users | Medium |

## 3. Findings by area

### 3.1 Performance — _generally excellent_

**Strengths**
- `src/App.tsx` lazy-loads every route; idle-time prefetches the core tabs (Quran, Prayer, Azkar) so they feel instant on first nav.
- `vite.config.ts` `manualChunks` cleanly splits framer-motion (`motion-vendor`), recharts/d3 (`charts-vendor`), Radix/cmdk/vaul (`ui-vendor`) and Supabase/idb/tanstack (`data-vendor`).
- `src/contexts/AudioPlayerContext.tsx:49–60` is split into State / Time / Ayah contexts. The 250 ms `timeupdate` tick only re-renders subscribers of the Time context, so the virtualized reader is unaffected.
- `src/lib/db.ts:160` keeps audio byte totals in `localStorage` instead of scanning every IDB blob — Storage Stats page stays cheap.

**Issues**

| Severity | File:Line | Issue | Fix direction |
|---|---|---|---|
| Medium | `src/lib/quran-api.ts`, `src/lib/tafsir-api.ts` | Fetches don't accept an `AbortSignal`; rapidly switching surahs leaves stale fetches in flight | Thread `AbortController` through `fetchSurahAyahs` / `fetchTafsir` and abort on unmount or surah change |
| Medium | `src/pages/SurahReaderPage.tsx:207` | `setAyahRef` is a no-op (`void el; void num;`); scroll-restoration to a remembered ayah relies on a missing registry | Store refs in a `Map` and use `virtualizer.scrollToIndex()` on restore |
| Low | `src/pages/SurahReaderPage.tsx:313` | `estimateSize: 170` is a flat average; very long ayahs cause visible jumps | Use `measureElement` (already supported by `@tanstack/react-virtual`) |
| Low | `src/contexts/LanguageContext.tsx` | Whole translations object lives in context; broad consumers re-render on language switch (rare, so low) | Memoize leaf consumers if it ever shows up in profiles |
| Low | `src/hooks/useReaderWakeLock.ts:69–73` | Listeners re-bind whenever `active` toggles | Hold the wake-lock sentinel in a ref and release before re-requesting |
| Low | `vite.config.ts` | Source maps not explicitly disabled for prod build | Set `build.sourcemap: false` (or `'hidden'` if you want them for Sentry) |

### 3.2 UI / UX & Accessibility

**Strengths**
- Solid RTL plumbing: `LanguageContext` flips `dir`, and `src/index.css` has `[dir="rtl"]`-specific decorative overrides.
- Touch targets and safe-area utilities exist (`.touch-target`, `.safe-area-bottom`, `.pb-nav-with-bar`).
- Modals/sheets use Radix → focus trapping and Escape handling for free.
- `AyahActionsSheet` programmatically focuses the note input when entering edit mode — nice screen-reader detail.

**Issues**

| Severity | File:Line | Issue | Fix direction |
|---|---|---|---|
| Medium | `src/index.css`, `tailwind.config.ts` | No `@media (prefers-reduced-motion: reduce)` block — heavy framer-motion animations play for users who asked the OS to disable motion | Global rule that sets `animation-duration: 0.01ms !important; transition-duration: 0.01ms !important;` |
| Medium | `src/pages/SurahReaderPage.tsx` (virtualized rows) | Ayah rows are flat `<div>`s — screen readers can't navigate by ayah | Render rows as `<article aria-label="Ayah {n}">` inside `<ol role="list">` |
| Medium | `src/components/layout/BottomNav.tsx` | `fixed bottom-3` doesn't lift fully past the iOS home indicator on some devices | Use `bottom: calc(0.75rem + env(safe-area-inset-bottom))` on the container |
| Low | "Gold" dark theme in `src/index.css` | `42 85% 55%` text on `24 10% 4%` background — borderline contrast | Bump to ≥ 60 % L or test with a contrast tool |
| Low | Various icon buttons | A few action icons rely on the icon alone with no `aria-label` | Audit `<button>`s wrapping `lucide-react` icons; add `aria-label` |
| Low | Loading/empty/error states | Inconsistent across pages — some use spinners, some show nothing while loading | Standardize on a `<PageState>` wrapper |

### 3.3 Offline & PWA

**Strengths**
- `vite-plugin-pwa` workbox config has dedicated `runtimeCaching` entries for tafsir, WBW, adhan audio, and both `api.alquran.cloud` and `api.quran.com`.
- `src/lib/quran-audio.ts` validates the magic bytes of downloaded audio so HTML error pages aren't cached as MP3s.
- `src/lib/syncQueue.ts` distinguishes retriable from permanent errors and drops the latter so the queue can't get stuck.
- `src/hooks/useServiceWorkerUpdate.ts` + `usePostUpdateChangelog.ts` give the user a clear "new version" prompt.

**Issues**

| Severity | File:Line | Issue | Fix direction |
|---|---|---|---|
| **High** | `src/lib/backup.ts` | Backup includes settings + counts but **not** the surah JSON, tafsir, or audio blobs; restore on a fresh device leaves the user with an empty offline library | Add an opt-in "include offline content" toggle that streams IDB blobs into a chunked archive; warn about size |
| Medium | `vite.config.ts` runtime cache for `api.alquran.cloud` & `api.quran.com` | `CacheFirst`, 30-day expiry — a corrected ayah translation can hide for a month | Switch to `StaleWhileRevalidate` with the same expiry |
| Medium | `src/lib/db.ts:85–123` | v6 → v7 migration calls `deleteObjectStore("bookmarks")` then re-imports from `localStorage` only; pure-IDB v6 data is lost | Migrate rows in place; only delete the store after a successful copy |
| Medium | `src/lib/db.ts:160` (byte tracker) | If a user clears `localStorage` but keeps IDB, the bytes counter desyncs and Settings reports 0 storage used | One-time "Recalculate storage" action in Settings → DevKit that scans IDB |
| Low | Cold-start without network | App boots fine, but there's no top-level "you're offline, last sync N ago" indicator | Tiny banner gated on `navigator.onLine === false` |

### 3.4 Audio & Listening

**Strengths**
- Three-tier context (State/Time/Ayah) keeps re-renders surgical.
- `src/lib/quran-audio.ts` validates magic bytes and times out at 90 s with `AbortController`.
- `mobileAudioManager` centralizes the iOS unlock dance.

**Issues**

| Severity | File:Line | Issue | Fix direction |
|---|---|---|---|
| **High** | `src/contexts/AudioPlayerContext.tsx:164` | Media Session "play" handler calls `mobileAudioManager.play("quran")` without re-resolving the source. After background suspension the `<audio>` element can lose its `src`, so lock-screen Play does nothing | Capture the current `orderedUrls[0]` (or surah/ayah) in a ref and re-set `audio.src` if empty before calling `play()` |
| Medium | `src/contexts/AudioPlayerContext.tsx:218–233` | Error listener runs through fallback URLs back-to-back; on flaky networks it burns through them in seconds | Add 1–2 s backoff between fallback attempts |
| Medium | `src/hooks/useGlobalAudioBootstrap.ts:15–33` | `warmAudioContext` runs on first interaction, but `primeAll` may not have unlocked every channel before the first play | Bind `primeAll` to the very first `pointerdown`/`touchstart` at the document level |
| Low | `src/hooks/useSleepModePlayer.ts` | Fade-out timer uses `setInterval`; if the tab is throttled the fade snaps | Use `audio.volume` ramp via `requestAnimationFrame` or Web Audio gain |
| Low | `src/hooks/useAdhan.ts` (Fajr-only adhan) | Switching mid-Fajr between full adhan and takbir-only doesn't always swap source until next call | Reset audio element when the per-prayer voice changes |

## 4. Phased remediation plan

Each phase is sized to fit a single focused work session. Items in a phase are independent unless noted.

### Phase 0 — Stop the bleeding (high-impact, low-risk; 1 sitting)

1. **Remove the hardcoded DevKit PIN.** Gate `/devkit` behind a Supabase auth check against an owner email allow-list. (`src/pages/DevKitPage.tsx`)
2. **Add a "claim local bookmarks" prompt on sign-in.** Default to "Decide later"; never silently merge. (`src/lib/bookmarks.ts`, `src/contexts/AuthContext.tsx`)
3. **Fix lock-screen Play.** Re-resolve audio source in the Media Session handler. (`src/contexts/AudioPlayerContext.tsx`)
4. **Switch API caches to `StaleWhileRevalidate`** so text/translation corrections actually reach users. (`vite.config.ts`)

### Phase 1 — Correctness in the spiritual core (1 sitting)

5. **Fix DST in prayer times** — compute timezone offset for the target date. (`src/lib/prayer-times.ts`)
6. **Fix Hifz streak DST drift** — diff date strings, not milliseconds. (`src/hooks/useHifzStreak.ts`)
7. **Normalize Alef variants** in recitation matching. (`src/lib/ayah-match.ts`)
8. **Persist `notifiedRef` for prayer notifications** so refresh doesn't re-fire. (`src/hooks/usePrayerNotifications.ts`)
9. **Move iOS DeviceOrientation permission behind a "Start Compass" button.** (`src/pages/QiblaPage.tsx`)
10. **Fix achievements stale-state bug** — pass stats as a dependency, not via `localStorage`. (`src/hooks/useAchievements.ts`)

### Phase 2 — Reliability & offline trust (1–2 sittings)

11. **Backup completeness** — opt-in "include offline content" with a chunked archive of IDB blobs. (`src/lib/backup.ts`)
12. **Safer IDB migration v6→v7** — copy then delete, don't delete then re-import. (`src/lib/db.ts`)
13. **Adhan reliability when backgrounded** — schedule via service-worker timer + `Notification`. (`src/hooks/useAdhan.ts`, `src/lib/notifications.ts`, SW)
14. **AbortController plumbing** in `quran-api.ts` and `tafsir-api.ts`; cancel on unmount/surah change.
15. **Fix `setAyahRef` no-op** so scroll-restoration to a remembered ayah works. (`src/pages/SurahReaderPage.tsx`)
16. **Speech-recognition restart backoff** to stop the rapid-fire loop. (`src/hooks/useSpeechRecognition.ts`)
17. **Idempotent sync writes** — convert remaining `insert` calls to `upsert` with a unique constraint. (`src/lib/syncQueue.ts`)
18. **Storage Stats "Recalculate"** action in DevKit/Settings to resync the `localStorage` byte tracker. (`src/lib/db.ts`)

### Phase 3 — UX polish & code hygiene (1 sitting)

19. **`prefers-reduced-motion` global rule.** (`src/index.css`)
20. **Semantic ayah rows** in the virtualized reader (`<article aria-label="…">` inside `<ol>`). (`src/pages/SurahReaderPage.tsx`)
21. **Bottom nav safe-area** lift on iOS via `env(safe-area-inset-bottom)`. (`src/components/layout/BottomNav.tsx`)
22. **Dark "Gold" theme contrast bump.** (`src/index.css`)
23. **"You're offline, last sync N ago" banner.** (new tiny component in `AppShell`)
24. **Hijri adjustment setting** for Ramadan/azkar dates. (`src/hooks/useRamadan.ts`, Settings)
25. **Pre-scheduled Friday reminder** Thursday night. (`src/hooks/useFridayReminders.ts`)
26. **Reading-goal debounce** + Tasbeeh UTC keying. (`src/hooks/useDailyReading.ts`, `src/pages/TasbeehPage.tsx`)
27. **Logger utility** + remove stray `console.log`s. (new `src/lib/logger.ts`)
28. **Re-enable unused-vars lint rule** and fix the fallout. (`eslint.config.js`)
29. **Unit tests** for `bookmarks.claimBookmarksForUser`, `syncQueue` flush race, `prayer-times` DST, `ayah-match` Arabic variants. (`src/lib/__tests__/`)
30. **Disable production source maps** in `vite.config.ts` (or set `'hidden'`).

### Phase 4 — Optional larger investments

- **Push notifications** for prayer/azkar so backgrounded PWAs are reliable on iOS 16.4+ (web push works in installed PWAs).
- **Server-authoritative timestamps** for sync to remove client clock-skew risk.
- **Consolidate `recitation_history` and `recitation_sessions`** into one table with a JSONB details column.

## 5. Out of scope / explicit non-goals

- Replacing Supabase with a self-hosted backend.
- Switching from npm to pnpm in CI (lockfile guard already covers both).
- Visual redesign — the audit found no design-level issues that warrant one.
- Any change to the Quran text data sources.
